Des téléchargements rangés sans y penser

Chaque fichier est enregistré dans un sous-dossier du répertoire de téléchargement par défaut — Images, Vidéos, Audios, Documents ou Autres — selon le type MIME et l’extension.

Build puis installation Comment ça marche

Fonctionnement

L’extension s’exécute en arrière-plan et intervient au moment où le navigateur choisit le nom du fichier : le chemin proposé devient NomDuDossier/nom-du-fichier.ext, toujours relatif au dossier de téléchargement configuré dans le navigateur.

La catégorisation combine le type MIME (quand il est disponible) et, à défaut, l’extension du fichier pour les cas ambigus.

Firefox : ce navigateur ne propose pas l’API onDeterminingFilename (réservée à Chromium). L’extension annule le premier téléchargement puis en relance un avec le même URL et le chemin cible — le résultat est le même, mais vous pouvez voir une entrée annulée brièvement dans le gestionnaire de téléchargements. Les liens blob: ou data: ne sont pas réécrits (limitation technique).

Build par navigateur

À la racine du dépôt (après git clone), installez les dépendances une fois : 

cd download-organizer
npm install

Ensuite, lancez exactement une des commandes ci-dessous selon le navigateur cible. WXT écrit le paquet prêt à charger dans .output/ (chemins relatifs à la racine du projet).

Chrome (MV3, cible par défaut)

npm run build

Dossier à charger : .output/chrome-mv3/

Microsoft Edge (MV3)

npm run build:edge

Dossier à charger : .output/edge-mv3/

Firefox (MV2)

npm run build:firefox

Dossier à charger : .output/firefox-mv2/

Archives pour publication (optionnel)

Pour générer un fichier .zip prêt à envoyer aux stores ou à partager : 

npm run zip              # Chrome (équivalent au build par défaut)
npm run zip:edge
npm run zip:firefox

Les archives sont créées sous .output/ (noms fournis par WXT dans le terminal).

Installation (mode développeur)

Après le build correspondant à votre navigateur, chargez le dossier de sortie indiqué ci-dessus (pas la racine du dépôt Git).

  • Chrome : chrome://extensions → Mode développeur → Charger l’extension non empaquetée → sélectionner .output/chrome-mv3
  • Edge : edge://extensions → Mode développeur → Charger l’extension non empaquetée → sélectionner .output/edge-mv3
  • Firefox : about:debugging#/runtime/this-firefox → Charger une extension temporaire → choisir le fichier manifest.json à l’intérieur de .output/firefox-mv2/

Développement (rechargement à chaud)

Le projet utilise WXT. En continu, sans relancer npm run build à la main : 

npm install
npm run dev

Chargez le dossier affiché dans le terminal (souvent .output/chrome-mv3 en dev). Variantes : npm run dev:firefox, npm run dev:edge.

Systèmes d’exploitation

L’extension suit le navigateur : sur Windows, macOS et Linux, le même paquet fonctionne dès que Chrome, Edge ou Firefox est installé. Le développement (Node.js, WXT) est également multiplateforme.
