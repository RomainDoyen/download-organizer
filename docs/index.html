Extension · Chrome · Edge · Firefox

Des téléchargements rangés sans y penser

Chaque fichier est enregistré dans un sous-dossier du répertoire de téléchargement par défaut — Images, Vidéos, Audios, Documents ou Autres — selon le type MIME et l’extension.

Installer Comment ça marche

Fonctionnement

L’extension s’exécute en arrière-plan et intervient au moment où le navigateur choisit le nom du fichier : le chemin proposé devient NomDuDossier/nom-du-fichier.ext, toujours relatif au dossier de téléchargement configuré dans le navigateur.

La catégorisation combine le type MIME (quand il est disponible) et, à défaut, l’extension du fichier pour les cas ambigus.

Installation (mode développeur)

Après avoir cloné le dépôt et lancé le build (voir la section Développement), chargez le dossier de sortie de l’extension :

  • Chrome : chrome://extensions → Mode développeur → Charger l’extension non empaquetée → dossier .output/chrome-mv3
  • Edge : edge://extensions → Charger l’extension non empaquetée → dossier .output/edge-mv3
  • Firefox : about:debugging → Chargement temporaire d’extensions → dossier .output/firefox-mv2

Développement

Le projet utilise WXT. Commandes utiles : 

npm install
npm run dev

Builds ciblés : npm run build, npm run build:firefox, npm run build:edge

Systèmes d’exploitation

L’extension suit le navigateur : sur Windows, macOS et Linux, le même paquet fonctionne dès que Chrome, Edge ou Firefox est installé. Le développement (Node.js, WXT) est également multiplateforme.
